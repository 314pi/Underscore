NAME _GetOpt -- Parse command line options and create environment vars SYNOPSIS CALL _GetOpt %* SET $NAME=x ::Parse options to current script CALL %$NAME% %* ::Show options set @x ::Clear env vars CALL %$NAME% DESCRIPTION If $NAME is not a defined environment var "UNDEFINED" is used as $NAME NOTE! If you entend to nest scripts both using %$NAME% you must start the nested script with a SETLOCAL statement to preserve the callers environment! EXAMPLES: If x.bat contains the statement: SET $NAME=x CALL %$NAME% %* SET @ :: Enable delayed expansion of vars to use content of vars SETLOCAL ENABLEDELAYEDEXPANSION ECHO z=!@%$NAME%.z! SETLOCAL DISABLEDELAYEDEXPANSION :: Or hard code reference ECHO a=%@x.z% :: Clear flags CALL %$NAME% Executing: x.bat -y:z -flag file -z:"hello world" x.bat will have the environment: $x.x=yy @x.z=1 @x.1=file @x.flag=1 @x.y=z @x.z="hello world" NOTE! Environment var DEBUG will enable debug info REQUIRES _Debug.cmd Setting up debug environment for batch scripts SOURCE C:\Users\Erik\Documents\GitHub\Underscore\_GetOpt.cmd (c)2015 Erik Bachmann, ClicketyClick.dk (ErikBachmann@ClicketyClick.dk)
